# Documentacao Do Projeto Quaz di Graca

Este diretorio concentra a documentacao-base para orientar planejamento, Codex, Claude Code, GitHub e implementacao.

## Arquivos

- [documento-mestre.md](documento-mestre.md): visao geral, produto, proposta de valor, MVP e stack.
- [arquitetura.md](arquitetura.md): estrutura tecnica, apps, infraestrutura e IA.
- [jornadas.md](jornadas.md): fluxos de consumidor, parceiro, admin e site publico.
- [banco-de-dados.md](banco-de-dados.md): tabelas iniciais, campos e regras de acesso.
- [roadmap.md](roadmap.md): fases de construcao do projeto.
- [decisoes.md](decisoes.md): decisoes tomadas, pontos a confirmar, riscos e mitigacoes.
- [supabase.md](supabase.md): configuracao do backend gratuito, env vars e migration de leads.
- [deploy.md](deploy.md): Vercel, Supabase, dominios, DNS e SEO tecnico.

## Como Usar Com Codex

Pedidos uteis:

```txt
Use quaz-brand, quaz-product e docs/documento-mestre.md para planejar a home.
Use quaz-architecture e docs/arquitetura.md para criar a estrutura do monorepo.
Use quaz-product e docs/jornadas.md para detalhar o fluxo do parceiro.
Use quaz-implementation e docs/banco-de-dados.md para iniciar o schema.
```
