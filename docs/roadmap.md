# Roadmap

## Fase 0: Base

Objetivo: preparar projeto para desenvolvimento.

Entregas:

- skills Codex criadas.
- documentacao base.
- GitHub privado.
- dominio definido.
- Cloudflare configurado.
- Vercel conectada.
- Supabase criado.

## Fase 1: Site Institucional

Objetivo: validar comunicacao e captar interessados.

Entregas:

- home.
- como funciona.
- seja parceiro.
- onde estamos.
- central de ajuda.
- termos e privacidade.
- formulario de lead parceiro.

## Fase 2: Web App Consumidor

Objetivo: permitir descoberta e compra/reserva.

Entregas:

- cadastro/login.
- localizacao.
- lista de achados.
- detalhe da loja.
- detalhe do Achado Quaz.
- checkout.
- pedido confirmado.
- meus pedidos.
- perfil.

## Fase 3: Painel Parceiro

Objetivo: permitir operacao pelos estabelecimentos.

Entregas:

- login parceiro.
- dashboard.
- dados da loja.
- criar/editar achados.
- pedidos recebidos.
- confirmar retirada.
- avaliacoes.

## Fase 4: Admin Interno

Objetivo: controlar a operacao.

Entregas:

- admin inicial de leads.
- aprovar parceiros.
- gerenciar lojas.
- gerenciar achados.
- ver pedidos.
- gerenciar suporte.
- cupons.
- cidades.

## Fase 5: Pagamentos E Piloto

Objetivo: validar transacao real.

Entregas:

- Mercado Pago.
- Pix.
- cartao.
- webhook.
- fluxo de cancelamento/reembolso.
- piloto em 1 cidade.

Meta de piloto:

```txt
10 a 30 parceiros
100 a 500 usuarios
primeiros pedidos reais
```

## Fase 6: App Mobile

Objetivo: melhorar experiencia recorrente.

Entregas:

- React Native + Expo.
- Android primeiro.
- iOS depois.
- push notifications.
- favoritos.
- geolocalizacao refinada.

## Fase 7: IA E Escala

Objetivo: automatizar operacao e crescer cidade por cidade.

Entregas:

- chatbot de suporte.
- descricoes automáticas para achados.
- sugestao de preco.
- resumo de avaliacoes.
- recomendacao personalizada.
- relatorios de impacto.
